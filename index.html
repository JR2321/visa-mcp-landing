CLI Tools

10 tools. Full payment lifecycle.

checkout_pay
amount, merchantName, description, preferredRail?, url?, method?, body?
Execute a payment. Auto-selects the optimal rail: card, USDC on Base, or Tempo pathUSD. Pass a URL to auto-pay 402 challenges.
checkout_get_status
no params
Enrollment status, card count, Base USDC balance, Tempo pathUSD balance, spending controls, and daily spend tracking.
checkout_get_cards
no params
List all enrolled payment cards with token IDs, brand, last4, expiry, and default status.
checkout_transaction_history
limit?
Recent transactions with amounts, merchants, rails, timestamps. Includes aggregate stats: total transactions, total spent, last 30 days.
checkout_update_spending_controls
maxTransactionAmount?, dailyLimit?, autoApproveUnder?
Set per-transaction max, daily spending cap, and auto-approve threshold. Payments under the threshold skip approval prompts.
checkout_add_card
no params
Opens VGS Collect enrollment form in browser. Card is tokenized by VGS (PCI DSS L1). Only aliases stored locally.
checkout_enroll_device
email?, phone?
Start Click to Pay enrollment. Sends a one-time verification code to email or phone.
checkout_verify_otp
otp, requestId
Complete enrollment with the verification code received from checkout_enroll_device.
checkout_login
no params
GitHub OAuth login followed by secure card enrollment. Opens browser for the full auth flow.
checkout_reset
confirm, retainCryptoKeys?
Clear enrollment and credentials. retainCryptoKeys (default true) keeps wallet keys so funds are safe.
CLI

Command line, the way it should be.

visa-cli
# Setup $ visa-cli init # Create wallet, initialize config $ visa-cli install claude # Add CLI server to Claude Code $ visa-cli status # Show enrollment and balances # Payments $ visa-cli pay <url> # Pay a merchant URL (auto-detect rail) $ visa-cli pay <url> --method POST --body '{"prompt":"hello"}' # Card enrollment $ visa-cli enroll-card # Open VGS Collect form in browser $ visa-cli enroll-card --demo # Demo mode, no VGS creds needed $ visa-cli cards # List enrolled cards
Payment Rails

Four rails. Auto-selected.

x402

Pay x402 endpoints with USDC on Base. Supports GET and POST. The native payment protocol for AI agents.

MPP / Tempo

Pay via Tempo network using pathUSD stablecoins. Machine Payments Protocol for instant settlement.

MPP / Card

Pay MPP endpoints with a VGS-vaulted card. Tokenized, PCI compliant, no PAN exposure.

Card (Direct)

Traditional card payments via VGS network tokenization. Works anywhere Visa is accepted online.

Security

Your agent pays. Your credentials never leave your machine.

Visa CLI is designed so your agent can make payments without ever seeing your card number, private key, or password. Every transaction requires your approval. Every credential stays on your device.

Card vault
VGS PCI L1
Cards tokenized by Very Good Security before they reach your machine. The agent only sees token aliases, never real card numbers.
Encryption
AES-256-GCM
Card data encrypted per-field with random IV. Decrypted only when you approve via biometric. The agent triggers the payment; you authorize it.
Credentials
OS Keychain
Stored in macOS Keychain, Windows Credential Manager, or Linux secret service. Never written to config files, env vars, or disk.
Approval
Biometric
Touch ID, Face ID, or Windows Hello required for every payment above your auto-approve threshold. No silent charges.
Spending controls
Policy Engine
Per-transaction caps, daily spending limits, and auto-approve thresholds. Your agent operates within boundaries you define.
Audit trail
Full History
Every transaction logged with amount, merchant, rail, and timestamp. Query your history anytime via CLI or MCP tool.
What the agent cannot do
✕ See your card number
✕ Export or transfer your private keys
✕ Modify its own spending limits
✕ Make payments without biometric approval
